## Step-by-Step Deployment Instructions

To run this architecture, you will deploy the **Backend (API)** to a free container host, and the **Frontend (UI)** to Vercel, using Vercel's proxy rewrite feature to connect them without CORS issues.

4. Configure the settings:

5. Under **Environment Variables**, add:
   - `DATABASE_URL`: `postgresql://neondb_owner:npg_yA2WZCTfjvV1@ep-still-mouse-aoxh3a9q-pooler.c-2.ap-southeast-1.aws.neon.tech/neondb?sslmode=require&channel_binding=require`

#### Option B: Deploying on Render (Free)
1. Go to [Render.com](https://render.com/) and sign up.
2. Click **New +** -> **Web Service**.
3. Link your GitHub repository `organ-donation`.
4. Configure the settings:
   - **Name**: `organ-donation-api`
   - **Root Directory**: (Leave blank)
   - **Runtime**: `Python`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn "backend.app:app" --bind 0.0.0.0:$PORT`
5. Click **Advanced**, and add your environment variables (`DATABASE_URL`, `SECRET_KEY`).
6. Click **Create Web Service**. Once deployed, copy your public URL (e.g. `https://organ-donation-api.onrender.com`).

### Step 2: Configure Vercel Proxy (vercel.json)

Your React frontend uses relative paths (like `/api/auth/login`) to make API requests. We will tell Vercel to route these `/api` requests directly to your newly deployed backend.

1. Open `frontend-react/vercel.json` in your editor.
2. Replace `https://YOUR_RAILWAY_HOSTNAME/` with your deployed backend URL from Step 1 (e.g., Koyeb or Render URL):
   ```json
   {
     "rewrites": [
       {
         "source": "/api/:path*",
         "destination": "https://organ-donation-xxx.koyeb.app/api/:path*"
       }
     ]
  }
  ```
3. Save, commit, and push this change to GitHub:
   ```bash
   git add frontend-react/vercel.json
   git commit -m "Configure vercel.json api proxy to point to cloud backend"
   git push origin main
   ```

### Step 3: Deploy Frontend on Vercel

1. Open your Vercel project settings for `organ-donation-o3u6`.
2. Under **Settings** -> **General** -> **Root Directory**:
   - Change it to **`frontend-react`**.
   - Click **Save**.
3. Under **Settings** -> **Build and Development settings**:
   - Framework Preset: **`Vite`** (detected automatically).
   - Build Command: **`npm run build`** (Vercel sets this automatically).
   - Output Directory: **`dist`** (Vercel sets this automatically).
4. Go to **Deployments**, click the three dots (`...`) on the latest commit you just pushed, and click **Redeploy**.

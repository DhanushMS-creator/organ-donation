### Option A: Deploy Python Backend to Vercel (Recommended - No Sleep, Free)

You can host both your React frontend and your Flask backend on Vercel as **two separate Vercel projects**.

1. Go to your Vercel Dashboard and click **Add New...** -> **Project**.
2. Import your `organ-donation` Git repository.
3. Configure the settings:
   - **Project Name**: `organ-donation-backend` (or similar)
   - **Root Directory**: Select **`backend`** (This tells Vercel to only build the python code inside the `backend` folder).
   - **Framework Preset**: Keep it as **Other** (Vercel will automatically detect Python and deploy it).
4. Under **Environment Variables**, add:
   - `DATABASE_URL`: `postgresql://neondb_owner:npg_yA2WZCTfjvV1@ep-still-mouse-aoxh3a9q-pooler.c-2.ap-southeast-1.aws.neon.tech/neondb?sslmode=require&channel_binding=require`

5. Click **Deploy**. Vercel will install dependencies from `backend/requirements.txt` and compile your API.
6. Once deployed, copy the backend deployment URL (e.g. `https://organ-donation-backend-xxx.vercel.app`).

### Option B: Deploy Python Backend to Render (Standard - Free)

1. Go to [Render.com](https://render.com/) and sign up with GitHub (no card required).
2. Click **New +** -> **Web Service**.
3. Link your GitHub repository `organ-donation`.
4. Configure the settings:
   - **Name**: `organ-donation-api`
   - **Root Directory**: (Leave blank)
   - **Runtime**: `Python`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn "backend.app:app" --bind 0.0.0.0:$PORT`
5. Click **Advanced**, and add your environment variables (`DATABASE_URL`, `SECRET_KEY`).
6. Click **Create Web Service**. Once deployed, copy your public URL (e.g. `https://organ-donation-api.onrender.com`).

### Step 2: Configure Vercel Proxy in Frontend

Your React frontend uses relative paths (like `/api/auth/login`) to make API requests. We will tell Vercel to route these `/api` requests directly to your newly deployed backend.

1. Open `frontend-react/vercel.json` in your local project.
2. Replace the destination URL with your deployed backend URL from Step 1 (either Vercel Backend URL from Option A or Render URL from Option B):
   ```json
   {
     "rewrites": [
       {
         "source": "/api/:path*",
         "destination": "https://YOUR_DEPLOYED_BACKEND_URL/api/:path*"
       }
     ]
  }
  ```
3. Save, commit, and push this change to GitHub:
   ```bash
   git add frontend-react/vercel.json
   git commit -m "Configure vercel.json api proxy to point to cloud backend"
   git push origin main
   ```
